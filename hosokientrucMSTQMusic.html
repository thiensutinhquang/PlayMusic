MSTQ Music App

Tài liệu này ghi chép lại cấu trúc dữ liệu, luồng hoạt động và các "thủ thuật cấp thấp" (Hack OS) được áp dụng để giữ ứng dụng chạy ngầm liên tục trên các thiết bị di động khắt khe nhất (như Xiaomi MIUI, iOS).

1. Tổng Quan Công Nghệ (Tech Stack)

React 18 (UMD)

Render UI toàn bộ ứng dụng mà không cần Node.js hay Webpack. Chạy trực tiếp từ CDN.

Tailwind CSS

Hệ thống Utility-classes quản lý giao diện, theme (Light/Dark/Lục bảo) bằng CSS Variables.

Native HTML5 Audio

Cốt lõi phát nhạc. Bị thao túng bởi JS để đánh lừa OS quản lý pin.

MediaSession API

Đồng bộ Tên bài hát, Ảnh bìa và Nút Next/Prev ra ngoài Màn hình khóa.

🔥 2. Lõi Âm Thanh Bất Tử (Heart of the System)

Đây là nơi chứa những dòng code đáng giá nhất của ứng dụng, giải quyết bài toán "Tắt màn hình bị HĐH giết App".

A. Thuật toán "Silence WakeLock" (Spotify Engine)

Sử dụng một thẻ id="mstq-wakelock" phát một đoạn nhạc Base64 câm (im lặng) lặp đi lặp lại vô tận. Điều này tạo ra một "luồng âm thanh ảo", báo cáo với Hệ điều hành rằng ứng dụng đang liên tục hoạt động, cấm HĐH đưa ứng dụng vào chế độ ngủ đông (Doze Mode).

<audio id="mstq-wakelock" loop playsinline style="display:none;">
   <source src="data:audio/mpeg;base64,SUQzBA..." type="audio/mpeg">
</audio>

B. Kiến trúc Ping-Pong (Double Engine Audio)

Sử dụng 2 thẻ Audio: Thẻ A và Thẻ B. Khi thẻ A đang hát bài 1, thẻ B sẽ bí mật "kéo mạng" (load) bài 2 vào RAM. Ngay khoảnh khắc thẻ A kết thúc, thẻ B lập tức hát lên.

Mục đích: Xóa bỏ độ trễ 3 giây chờ tải mạng giữa 2 bài hát. Không có khoảng im lặng -> HĐH không có lý do để giết tiến trình. 

const activePlayerIdRef = useRef('a'); // Con trỏ xác định thẻ nào đang nắm quyền
// Khi bài kết thúc:
const newActive = e.target.id === 'mstq-player-a' ? pB : pA;
activePlayerIdRef.current = newActive.id === 'mstq-player-a' ? 'a' : 'b';
newActive.play(); // Kêu ngay tức thì vì đã được Preload

C. WebAudio Bypass (Bảo vệ Mobile)

WebAudio API (`AudioContext`) dùng để làm mượt âm lượng, nhưng nó là "kịch độc" trên Mobile. iOS/Android sẽ tự động Suspend AudioContext khi tắt màn hình. 

const isMobileOS = /iPhone|iPad|iPod|Android/i.test(navigator.userAgent);
const ensureAudioContext = async () => {
    if (isMobileOS) return; // BYPASS TOÀN BỘ WEBAUDIO NẾU LÀ ĐIỆN THOẠI
    // ... logic web audio cho PC
}

3. Sơ Đồ Quản Lý Trạng Thái (State & Memory)

Ứng dụng sử dụng React useState cho UI, nhưng sử dụng useRef cho các luồng xử lý Âm thanh ngầm để tránh lỗi bất đồng bộ (Race Condition).

State / Ref Name LocalStorage Key Chức năng cốt lõi
playlist
playlistRef		 mstq_songs_v18 Chứa mảng các object bài hát. Ref dùng để cho luồng C++ Native gọi tới mà không bị lỗi Stale Closure.
currentIndex
currentIndexRef		 Nằm trong mstq_resume_v22 Vị trí bài hát hiện tại. Phải đồng bộ cả State (để UI sáng viền) và Ref (để hàm Play lấy đúng URL).
sourcePref mstq_sources_pref Lưu cài đặt Nguồn tải nhạc (Bật Archive, nguồn tự chọn M3U).
lastMediaSessionUpdateRef - (Không lưu) Biến cực kỳ quan trọng. Giới hạn việc cập nhật Màn hình khóa chỉ 1 lần/giây (Tối ưu 95% CPU khi chạy nền).

4. Phân rã Component UI (React)

<StartGate />

Popup đếm ngược 4s đầu tiên. Đóng vai trò Mở khóa (Unlock) quyền tương tác âm thanh của trình duyệt mà không cần người dùng thao tác.

<TopBar />

Thanh điều hướng cố định trên cùng. Chứa chức năng đổi Theme và Dropdown Menu "Liên kết" (chống che khuất Z-index = 100).

<HomeTab />

Render danh sách `playlist`. Sử dụng kỹ thuật cắt mảng (slice) thông qua biến `visible` để tối ưu DOM ban đầu (không render 1000 bài cùng lúc). Có auto-scroll tới bài đang phát.

<MiniPlayer /> & <NowPlaying />

MiniPlayer nổi dưới đáy. Khi click mở ra NowPlaying toàn màn hình. Điều khiển chức năng Next/Prev/Play/Tua nhạc.

5. PWA Engine V29 (Hệ thống Cài đặt Thông minh)

Đoạn script JS thuần nằm dưới cùng file HTML, hoạt động hoàn toàn độc lập với React để bắt các sự kiện cấp hệ điều hành.

  • Phát hiện In-App
    Nếu người dùng mở link từ Zalo/Facebook, trình duyệt con (Webview) này không hỗ trợ nhạc nền. Engine sẽ hiện Modal đỏ cảnh báo người dùng bấm 3 chấm -> Mở bằng Chrome/Safari.
  • Cài đặt PC/Android
    Bắt sự kiện `beforeinstallprompt`. Nếu người dùng dùng Chrome, nút [Tải App] (FAB) sẽ hiện lên dưới góc phải. Bấm vào sẽ gọi lệnh cài đặt PWA tự động.
  • Nhận diện iOS
    Safari iOS không cho cài tự động. Engine sẽ hiện Modal hướng dẫn từng bước: "Bấm nút Chia sẻ -> Thêm vào MH chính" kèm hiệu ứng mũi tên nhấp nháy chỉ đúng vị trí nút ở dưới đáy màn hình.

  • 2. KHÔNG DÙNG State (`currentIndex`) bên trong hàm `onEndedNative`:
